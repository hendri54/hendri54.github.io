    Code Loading¶

    When you write a file with some code, how does Julia know where to find it?

    From the REPL include(/path/to/file.jl) does the trick. This has the same effect as typing the code in the REPL. The same can be done inside another file.

    But what if you have a big chunk of code that you want to be reusable? What if you want to use someone else's code? This is where packages come into play.

    Packages¶

    A package is a directory with code that declares all of its dependencies. It is self-contained.

    The package MyPackage defines the module MyPackage plus types and functions.

    To use a package, write using MyPackage and voila - all the types and functions defined in MyPackage are available in your code, including the code MyPackage requires from other packages.

    How does Julia find packages?

    Many packages are registered in the General Registry. For them, Pkg.add("MyPackage") figures out where to get the code so that using MyPackage works.

    For unregistered packages, such as your own code (for now), you can make the code available by "devloping" the packages with Pkg.develop(/path/to/MyPackage).

    There is one fundamental difference between add and develop.

    • add fixes the version of the package until you manually Pkg.update("MyPackage"). Even if the developer changes the code, the version that you are using remains unchanged.
    • develop tells Julia to track whatever code changes happen in the directory where the package code resides (the one you provide with the develop command).

    The LOAD_PATH¶

    You can also load modules that are not packaged as packages by putting their directories into the LOAD_PATH environment variable.

    We will not use this approach. It is easier to package everything as a package instead.

    Environments¶

    This is a key concept for Julia!

    An environment is a directory that contains a Project.toml file. This file declares all of the dependencies that are available when the environment is "activated".

    Whe you start Julia, you are in the v.1.5 environment. In my case, it declares that the following packages are available:

    julia> using Pkg

julia> Pkg.status()
Status `~/.julia/environments/v1.5/Project.toml`
  [6e4b80f9] BenchmarkTools v0.5.0
  [89398ba2] LocalRegistry v0.3.2
  [5fb14364] OhMyREPL v0.5.10
  [d254efa0] PkgSkeleton v0.4.0
  [14b8a8f1] PkgTemplates v0.7.14
  [295af30f] Revise v3.1.12

    Tip: keep this environment minimal. Here you just want to list packages that you use while developing your code (e.g. Revise.jl).

    If you now activate a different directory using Pkg.activate("/path/to/dir"), additional packages become available. This is known as stacked environments. They are stacked in the sense that activating another environment retains the packages that are already activated. This is the reason why you want your v.1.5 environment to be minimal.

    Note that a package is just an environment that satisfies some additional criteria (e.g., a specific directory structure is present).
